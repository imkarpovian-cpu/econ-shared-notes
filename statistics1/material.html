第1章 基本操作（算術・オブジェクト・コメント）

📌 要点

  • Rは電卓のように算術演算ができる（+ - * / ^）
  • # 以降はコメント（実行されない）
  • 計算結果は <-（代入演算子）でオブジェクトに保存できる
  • オブジェクト名の大文字・小文字は区別される（result と Result は別物）
  • オブジェクトは上書き可能

📖 一般形

算術演算
数値 + 数値    # 加算
数値 - 数値    # 減算
数値 * 数値    # 乗算
数値 / 数値    # 除算
数値 ^ 数値    # べき乗
sqrt(数値)     # 平方根
abs(数値)      # 絶対値
コメント
コード    # コメント（この部分は実行されない）
オブジェクトへの代入
オブジェクト名 <- 値

🧮 具体例

例 
5 + 3          # → 8
5 ^ 3          # → 125
sqrt(4)        # → 2
result <- 5 + 3  # resultに8が保存される
result         # → 8

⚠️ つまずきポイント

注意 大文字・小文字は区別される。result と Result は別のオブジェクト。未定義のオブジェクトを参照すると Error: object 'Result' not found になる。
注意 10^20 は 1e+20、1 / 9^50 は 1.940325e-48 のように表示される。e+N は $10^N$ を意味する。

第2章 ベクトルとデータ型

📌 要点

  • ベクトルは c() で作成。同じデータ型の要素を複数持つ
  • インデックスは 1始まり。x[2] で2番目の要素
  • 主なデータ型：numeric（数値）、character（文字）、logical（論理）
  • ベクトル内で型が混在すると自動的に型変換される（数値→文字）
  • 比較演算子 == != > < は論理値を返す
  • 論理値は数値演算時に TRUE=1、FALSE=0 に変換される

📖 一般形

ベクトルの作成
c(要素1, 要素2, ...)
ベクトルの結合
c(ベクトル1, ベクトル2)
要素の取り出し
ベクトル[インデックス]          # 1つ取り出し
ベクトル[c(i, j)]           # 複数取り出し
ベクトル[-c(i, j)]          # 指定を除外
ベクトル["名前"]             # 名前で取り出し
ベクトルの演算（要素ごと）
ベクトル / 数値              # 全要素を割る
ベクトル / ベクトル[1]        # 最初の要素で基準化
ベクトルに名前をつける
names(ベクトル) <- 名前のベクトル
数列の生成
seq(from = 開始, to = 終了, by = 間隔)
開始:終了    # 1刻みの連続整数
データ型の確認・変換・判定
一般形説明
class(x)xのデータ型を返す
as.numeric(x)xを数値型に変換
as.factor(x)xを因子型に変換
as.integer(x)xを整数型に変換（TRUE→1, FALSE→0）
is.numeric(x)xが数値型ならTRUE
is.character(x)xが文字型ならTRUE
is.na(x)xが欠損値(NA)ならTRUE
比較演算子
演算子意味返り値
==等しいTRUE / FALSE
!=等しくない（≠）TRUE / FALSE
>より大きいTRUE / FALSE
<より小さいTRUE / FALSE
>=以上TRUE / FALSE
<=以下TRUE / FALSE
論理演算子（条件の組み合わせ）
演算子意味
&かつ（AND）
|または（OR）
!否定（NOT）

🧮 具体例

例：ベクトルの作成と操作 
world.pop <- c(2525779, 3026003, 3691173)
world.pop[2]                # → 3026003
world.pop[c(1, 3)]          # → 2525779 3691173
pop.million <- world.pop / 1000  # 各要素を1000で割る
例：数列 
seq(from = 1950, to = 2010, by = 10)  # → 1950 1960 ... 2010
2008:2012                   # → 2008 2009 2010 2011 2012
例：論理値の数値演算 
x <- c(TRUE, FALSE, TRUE)
mean(x)    # → 0.6667（TRUEの割合）
sum(x)     # → 2（TRUEの個数）

⚠️ つまずきポイント

注意 ベクトル内で型が混在すると、数値が文字に自動変換される。
c(1, 2, "a") → "1" "2" "a"（全部文字型になる）
注意 文字型の "3.14" と数値型の 3.14 は別物。"3.14" + 1 はエラーになる。as.numeric("3.14") で変換が必要。
注意 文字列の比較でも大文字・小文字は区別される。"Hello" == "hello" は FALSE。

第3章 関数の基本と自作関数

📌 要点

  • 関数は特定の入力（引数）に対して特定の出力を返すもの
  • 引数は位置（順序）でも、名前でも指定できる
  • 引数にはデフォルト値を設定できる
  • ?関数名 でヘルプを確認できる
  • function(){} で自作関数を定義できる

📖 一般形

組み込み関数（1変数）
一般形説明
length(x)ベクトルの要素数
min(x)最小値
max(x)最大値
range(x)範囲（最小値と最大値）
mean(x)平均値
median(x)中央値
sum(x)合計値
sd(x)標準偏差
var(x)分散
summary(x)要約統計量（最小・四分位・最大・平均）
log(x)自然対数（底は $e$）
組み込み関数（多変数）
一般形説明
rep(x, times)xをtimes回繰り返すベクトル
paste(x, y, sep = "区切り")文字列の結合
quantile(x, probs = 確率ベクトル)指定した分位数を計算
cor(x, y)相関係数
NA対応引数
mean(x, na.rm = TRUE)    # NAを除外して計算
sd(x, na.rm = TRUE)
var(x, na.rm = TRUE)

NAを含むベクトルにそのまま mean(x) すると結果も NA になる

関数のヘルプ
?関数名    # RStudio右下のペインにヘルプが表示
自作関数の定義
関数名 <- function(引数1, 引数2, ...) {
  処理
  return(出力)
}
デフォルト引数付き関数
関数名 <- function(引数1, 引数2 = デフォルト値) {
  処理
  return(出力)
}
引数の指定方法
関数(値1, 値2)                # 位置（順序）で指定
関数(引数名1 = 値1, 引数名2 = 値2)  # 名前で指定（順序自由）

🧮 具体例

例：自作関数の定義と使用 
my.difference <- function(input1, input2) {
  output <- input1 - input2
  return(output)
}
my.difference(1, 3)            # → -2
my.difference(input2 = 3, input1 = 1)  # → -2（名前指定なら順序自由）
例：記述統計の自作関数 
my.summary <- function(x) {
  s.out <- sum(x)
  l.out <- length(x)
  m.out <- s.out / l.out
  out <- c(s.out, l.out, m.out)
  names(out) <- c("sum", "length", "mean")
  return(out)
}

⚠️ つまずきポイント

注意 na.rm = TRUE を忘れると、NAが1つでもあれば結果がNAになる。
mean(c(1, 2, NA)) → NA
mean(c(1, 2, NA), na.rm = TRUE) → 1.5
注意 log(x) は自然対数（底は $e = 2.71828...$）。0や負の値には適用できない（-Inf や NaN を返す）。

第4章 データフレーム操作（読み込み・加工・保存）

📌 要点

  • tidyverse（readrなど）を使う現代的な書き方が授業推奨
  • パイプ演算子 |> でコードを上から下に読める
  • filter() で行の選択、select() で列の選択
  • mutate() で新しい列の追加・変換
  • pivot_longer() で横→縦変換、pivot_wider() で縦→横変換
  • 整然データ（tidy data）: 各変数＝列、各観測＝行、各値＝セル

📖 一般形

パッケージの導入
install.packages("パッケージ名")    # 初回のみ
library(パッケージ名)              # 毎回読み込み
データの読み込み
データ名 <- read_csv("ファイルパス.csv")
パイプ演算子
x |> f()    # f(x) と同じ意味
x |> f() |> g()    # g(f(x)) と同じ意味

%>% と |> は基本的に同じ。授業では |> を使用。

行の選択（filter）
データ |> filter(条件式)
データ |> filter(条件1 & 条件2)    # AND
データ |> filter(条件1 | 条件2)    # OR
データ |> filter(列名 %in% c("値1", "値2"))  # 複数値のいずれか
データ |> filter(!is.na(列名))    # NAを除外
列の選択（select）
データ |> select(列名1, 列名2)          # 指定列を残す
データ |> select(-c(列名1, 列名2))     # 指定列を除外
列の追加・変換（mutate）
データ |> mutate(
  新列名 = 計算式,
  新列名2 = as.factor(既存列),
  新列名3 = ifelse(条件, TRUEの値, FALSEの値)
)
ifelse関数
ifelse(条件, TRUEのときの値, FALSEのときの値)
列名の確認・変更
colnames(データ)              # 列名を確認
colnames(データ) <- 新しい名前のベクトル  # 列名を変更
データの形状変換
# 横→縦（ロング形式に変換）
データ |> pivot_longer(
  cols = 変換する列の指定,    # everything() で全列
  names_to = "新しい列名（元の列名が入る）",
  values_to = "新しい列名（値が入る）"
)

# 縦→横（ワイド形式に変換）
データ |> pivot_wider(
  names_from = 列名にする列,
  values_from = 値にする列
)
並び替え
データ |> arrange(列名)     # 昇順
データ |> arrange(desc(列名))  # 降順
重複の除去
データ |> distinct(列名)    # 指定列の一意な値を返す
値の取り出し
データ |> pull(列名)       # 列をベクトルとして取り出す
データ$列名                  # 同じ効果（Base R記法）
データ$列名[行番号]          # 特定の行の値
データフレームの結合
一般形説明
inner_join(x, y, by = "列名")共通の値が一致する行のみ結合
left_join(x, y, by = "列名")左側を全て保持し、右側をマッチ
right_join(x, y, by = "列名")右側を全て保持し、左側をマッチ
full_join(x, y, by = "列名")両方の全行を保持
データの保存
write_csv(データ, "パス/ファイル名.csv")    # CSVとして保存（型情報なし）
saveRDS(データ, "パス/ファイル名.rds")      # RDS形式で保存（型情報あり）
作業ディレクトリ・ファイル操作
getwd()                      # 現在の作業ディレクトリを確認
list.files("フォルダパス")     # フォルダ内のファイル一覧

⚠️ つまずきポイント

注意 列名にスペースが含まれる場合はバッククォートで囲む。
filter(`Country Name` == "Japan")
注意 CSVは型情報を保持しない。RDSは型情報を保持する。大きなデータではparquet形式が推奨。

第5章 データ集計（group_by・summarize・table）

📌 要点

  • group_by() |> summarize() はtidyverseでのグループ別集計の基本パターン
  • ダミー変数（0/1）の平均値 ＝ 条件を満たす割合
  • 論理条件の mean() も割合を返す（TRUE=1, FALSE=0）
  • table() はクロス集計表を作成
  • sapply() で複数列に関数を一括適用

📖 一般形

グループ別集計（最重要パターン）
データ |>
  group_by(グループ列) |>
  summarize(
    新列名 = mean(列名),    # 平均
    新列名 = sum(列名),     # 合計
    新列名 = median(列名)   # 中央値
  )
複数変数でグループ化
データ |>
  group_by(列名1, 列名2) |>
  summarize(新列名 = 集計関数(列名))
ダミー変数の平均 ＝ 割合（数式）

ダミー変数 $x_i \in \{0, 1\}$ について:

$$\bar{x} = \frac{1}{n}\sum_{i=1}^{n} x_i = \frac{k}{n}$$

$k$ は条件を満たす($x_i = 1$)データの数。つまり mean(call) = 書類通過率。

論理条件の割合を計算
mean(列名 == "値")        # "値"と一致する割合
mean(列名 < 数値)         # 数値未満の割合
mean(is.na(列名))        # 欠損の割合
sum(is.na(列名))         # 欠損の個数
クロス集計表
データ |>
  select(列名1, 列名2) |>
  table()                   # クロス集計

addmargins(集計表)          # 行・列の合計を追加
prop.table(集計表)          # 比率に変換

table(exclude = NULL)      # NAも集計に含める
sapply（複数列に関数を適用）
データ |>
  sapply(
    関数名,
    追加引数...
  )
十分位数の計算
データ |>
  select(列名1, 列名2) |>
  sapply(quantile, probs = seq(0, 1, by = 0.1), na.rm = TRUE)

🧮 具体例

例：人種別書類通過率 
resume |>
  group_by(race) |>
  summarize(
    call_rate = mean(call),   # 通過率
    call_count = sum(call)    # 通過数
  )
例：差の差分法（DID） 
did <- minwage |>
  group_by(state) |>
  summarize(
    fullBefore_prop = mean(fullBefore_prop),
    fullAfter_prop = mean(fullAfter_prop)
  ) |>
  mutate(diff_before_after = fullAfter_prop - fullBefore_prop)
# DID推定値:
did$diff_before_after[1] - did$diff_before_after[2]

⚠️ つまずきポイント

注意 group_by() は見た目を変えないが、以降の summarize() がグループごとに計算される。# Groups: の表示で確認可能。
注意 table() はデフォルトでNAを無視する。NAも集計に含めたい場合は table(exclude = NULL) を使う。

第6章 ggplotによるグラフ作成

📌 要点

  • ggplotの基本構造: ggplot(aes()) + geom_xxx() + labs() + theme()
  • グラフの種類: 棒グラフ(geom_bar)、ヒストグラム(geom_histogram)、箱ひげ図(geom_boxplot)、散布図(geom_point)、線グラフ(geom_line)
  • aes() でデータと見た目（x軸・y軸・色・形）の対応を指定
  • ggsave() でグラフを画像として保存

📖 一般形

ggplotの基本構造
データ |>
  ggplot(aes(x = x軸の列, y = y軸の列)) +
  geom_xxx() +
  labs(x = "x軸ラベル", y = "y軸ラベル", title = "タイトル") +
  theme_minimal(base_family = "フォント名")
棒グラフ
geom_bar(stat = "identity")    # y軸の値をそのまま使う

stat = "identity" を指定すると、データの値をそのまま棒の高さに使う。

ヒストグラム
ggplot(aes(x = 列名, y = after_stat(density))) +
geom_histogram(
  binwidth = ビンの幅,
  boundary = ビンの境界値,
  color = "枠線色",
  fill = "塗り色"
)

after_stat(density): y軸を密度にする。binwidth: 各ビンの幅。boundary: ビンの開始位置。

箱ひげ図
ggplot(aes(x = グループ列, y = 数値列)) +
geom_boxplot(
  fill = "塗り色",
  color = "枠線色",
  outlier.shape = 数値,    # 外れ値の形状
  outlier.size = 数値,     # 外れ値のサイズ
  outlier.alpha = 数値     # 外れ値の透明度
)

箱ひげ図の要素: 最小値、第1四分位数(Q1)、中央値、第3四分位数(Q3)、最大値。

散布図
ggplot(aes(x = 列名1, y = 列名2, color = グループ列, shape = グループ列)) +
geom_point(
  size = 点のサイズ,
  alpha = 透明度(0〜1)
)
線グラフ
ggplot(aes(x = 列名1, y = 列名2, color = グループ列, linetype = グループ列)) +
geom_line(linewidth = 線の太さ)
色・形状の手動設定
scale_color_manual(
  name = "凡例タイトル",
  labels = c("元の値" = "表示名", ...),
  values = c("元の値" = "色コード", ...)
)

scale_shape_manual(
  name = "凡例タイトル",
  values = c("元の値" = 形状番号, ...)
)

scale_color_brewer(palette = "パレット名")  # ColorBrewerのパレット
軸のカスタマイズ
scale_x_continuous(limits = c(最小, 最大), breaks = 目盛りベクトル)
scale_x_discrete(labels = c("ラベル1", "ラベル2", ...))  # 離散軸のラベル変更
グラフの保存
ggsave("フォルダ/ファイル名.png")

⚠️ つまずきポイント

注意 ggplotの各要素は + でつなぐ（パイプ |> ではない）。
注意 日本語フォントの指定: Macでは "Hiragino Sans"、Windowsでは "Yu Gothic" を使う。
注意 ggsave() はフォルダが存在しないとエラー。事前に dir.create("figure") が必要。

第7章 統計分析（相関・k平均法・Q-Qプロット）

📌 要点

  • 相関係数 cor(x, y) は2変数の線形関係の強さ（-1〜1）
  • Q-Qプロットは2つの分布の同一性を視覚的に確認する方法
  • $k$平均法は、データを$k$個のクラスターに分けるアルゴリズム
  • 対数変換 log() は分布の偏りを修正し、差を変化率に変換する

📖 一般形

相関係数
データ |>
  summarise(相関名 = cor(列名1, 列名2))
Q-Qプロット
qqplot(
  ベクトル1,
  ベクトル2,
  xlab = "x軸ラベル",
  ylab = "y軸ラベル",
  main = "タイトル",
  xlim = c(最小, 最大),
  ylim = c(最小, 最大)
)
abline(切片, 傾き, lwd = 線の太さ)  # 45度線: abline(0, 1)
k平均法
結果 <- データ |>
  select(変数1, 変数2) |>
  kmeans(centers = k, nstart = 試行回数)

結果$centers     # 各クラスターの中心点
結果$cluster     # 各データポイントのクラスター番号
クラスターと政党のクロス集計
table(
  party = データ$列名,
  cluster = 結果$cluster
)
自然対数変換
log(x)    # 底はe（ネイピア数）

対数変換の性質: $\log(x_2) - \log(x_1) = \log\left(\frac{x_2}{x_1}\right) \approx \frac{x_2 - x_1}{x_1}$（変化率への変換）

🧮 具体例

例：k平均法の実行と結果確認 
result <- congress |>
  filter(congress == 112) |>
  select(dwnom1, dwnom2) |>
  kmeans(centers = 2, nstart = 5)

result$centers    # 中心点の座標
table(
  party = congress$party[congress$congress == 112],
  cluster = result$cluster
)  # クラスターと政党の対応

⚠️ つまずきポイント

注意 $k$平均法のクラスター番号（1, 2, 3...）に意味はない。実行するたびに番号と中心点の対応が変わりうる。nstartで複数回試行し最良の結果を選ぶ。
注意 log(0) は -Inf、log(-1) は NaN を返す。対数変換する前にデータに0や負の値がないか確認すること。
注意 Q-Qプロットで点が45度線上にあれば2つの分布は同じ。線より上＝y側の方が大きい、線より下＝x側の方が大きい。
