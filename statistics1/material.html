第1章 基本操作（算術・オブジェクト・コメント）

📌 要点

  • Rは電卓のように算術演算ができる（+ - * / ^）
  • # 以降はコメント（実行されない）
  • 計算結果は <-（代入演算子）でオブジェクトに保存できる
  • オブジェクト名の大文字・小文字は区別される（result と Result は別物）
  • オブジェクトは上書き可能

📖 一般形

算術演算
数値 + 数値    # 加算
数値 - 数値    # 減算
数値 * 数値    # 乗算
数値 / 数値    # 除算
数値 ^ 数値    # べき乗
sqrt(数値)     # 平方根
abs(数値)      # 絶対値
コメント
コード    # コメント（この部分は実行されない）
オブジェクトへの代入
オブジェクト名 <- 値

🧮 具体例

例 
5 + 3          # → 8
5 ^ 3          # → 125
sqrt(4)        # → 2
result <- 5 + 3  # resultに8が保存される
result         # → 8

⚠️ つまずきポイント

注意 大文字・小文字は区別される。result と Result は別のオブジェクト。未定義のオブジェクトを参照すると Error: object 'Result' not found になる。
注意 10^20 は 1e+20、1 / 9^50 は 1.940325e-48 のように表示される。e+N は $10^N$ を意味する。

第2章 ベクトルとデータ型

📌 要点

  • ベクトルは c() で作成。同じデータ型の要素を複数持つ
  • インデックスは 1始まり。x[2] で2番目の要素
  • 主なデータ型：numeric（数値）、character（文字）、logical（論理）
  • ベクトル内で型が混在すると自動的に型変換される（数値→文字）
  • 比較演算子 == != > < は論理値を返す
  • 論理値は数値演算時に TRUE=1、FALSE=0 に変換される

📖 一般形

ベクトルの作成
c(要素1, 要素2, ...)
ベクトルの結合
c(ベクトル1, ベクトル2)
要素の取り出し
ベクトル[インデックス]          # 1つ取り出し
ベクトル[c(i, j)]           # 複数取り出し
ベクトル[-c(i, j)]          # 指定を除外
ベクトル["名前"]             # 名前で取り出し
ベクトルの演算（要素ごと）
ベクトル / 数値              # 全要素を割る
ベクトル / ベクトル[1]        # 最初の要素で基準化
ベクトルに名前をつける
names(ベクトル) <- 名前のベクトル
数列の生成
seq(from = 開始, to = 終了, by = 間隔)
開始:終了    # 1刻みの連続整数
データ型の確認・変換・判定
一般形説明
class(x)xのデータ型を返す
as.numeric(x)xを数値型に変換
as.factor(x)xを因子型に変換
as.integer(x)xを整数型に変換（TRUE→1, FALSE→0）
is.numeric(x)xが数値型ならTRUE
is.character(x)xが文字型ならTRUE
is.na(x)xが欠損値(NA)ならTRUE
比較演算子
演算子意味返り値
==等しいTRUE / FALSE
!=等しくない（≠）TRUE / FALSE
>より大きいTRUE / FALSE
<より小さいTRUE / FALSE
>=以上TRUE / FALSE
<=以下TRUE / FALSE
論理演算子（条件の組み合わせ）
演算子意味
&かつ（AND）
|または（OR）
!否定（NOT）

🧮 具体例

例：ベクトルの作成と操作 
world.pop <- c(2525779, 3026003, 3691173)
world.pop[2]                # → 3026003
world.pop[c(1, 3)]          # → 2525779 3691173
pop.million <- world.pop / 1000  # 各要素を1000で割る
例：数列 
seq(from = 1950, to = 2010, by = 10)  # → 1950 1960 ... 2010
2008:2012                   # → 2008 2009 2010 2011 2012
例：論理値の数値演算 
x <- c(TRUE, FALSE, TRUE)
mean(x)    # → 0.6667（TRUEの割合）
sum(x)     # → 2（TRUEの個数）

⚠️ つまずきポイント

注意 ベクトル内で型が混在すると、数値が文字に自動変換される。
c(1, 2, "a") → "1" "2" "a"（全部文字型になる）
注意 文字型の "3.14" と数値型の 3.14 は別物。"3.14" + 1 はエラーになる。as.numeric("3.14") で変換が必要。
注意 文字列の比較でも大文字・小文字は区別される。"Hello" == "hello" は FALSE。

第3章 関数の基本と自作関数

📌 要点

  • 関数は特定の入力（引数）に対して特定の出力を返すもの
  • 引数は位置（順序）でも、名前でも指定できる
  • 引数にはデフォルト値を設定できる
  • ?関数名 でヘルプを確認できる
  • function(){} で自作関数を定義できる

📖 一般形

組み込み関数（1変数）
一般形説明
length(x)ベクトルの要素数
min(x)最小値
max(x)最大値
range(x)範囲（最小値と最大値）
mean(x)平均値
median(x)中央値
sum(x)合計値
sd(x)標準偏差
var(x)分散
summary(x)要約統計量（最小・四分位・最大・平均）
log(x)自然対数（底は $e$）
組み込み関数（多変数）
一般形説明
rep(x, times)xをtimes回繰り返すベクトル
paste(x, y, sep = "区切り")文字列の結合
quantile(x, probs = 確率ベクトル)指定した分位数を計算
cor(x, y)相関係数
NA対応引数
mean(x, na.rm = TRUE)    # NAを除外して計算
sd(x, na.rm = TRUE)
var(x, na.rm = TRUE)

NAを含むベクトルにそのまま mean(x) すると結果も NA になる

関数のヘルプ
?関数名    # RStudio右下のペインにヘルプが表示
自作関数の定義
関数名 <- function(引数1, 引数2, ...) {
  処理
  return(出力)
}
デフォルト引数付き関数
関数名 <- function(引数1, 引数2 = デフォルト値) {
  処理
  return(出力)
}
引数の指定方法
関数(値1, 値2)                # 位置（順序）で指定
関数(引数名1 = 値1, 引数名2 = 値2)  # 名前で指定（順序自由）

🧮 具体例

例：自作関数の定義と使用 
my.difference <- function(input1, input2) {
  output <- input1 - input2
  return(output)
}
my.difference(1, 3)            # → -2
my.difference(input2 = 3, input1 = 1)  # → -2（名前指定なら順序自由）
例：記述統計の自作関数 
my.summary <- function(x) {
  s.out <- sum(x)
  l.out <- length(x)
  m.out <- s.out / l.out
  out <- c(s.out, l.out, m.out)
  names(out) <- c("sum", "length", "mean")
  return(out)
}

⚠️ つまずきポイント

注意 na.rm = TRUE を忘れると、NAが1つでもあれば結果がNAになる。
mean(c(1, 2, NA)) → NA
mean(c(1, 2, NA), na.rm = TRUE) → 1.5
注意 log(x) は自然対数（底は $e = 2.71828...$）。0や負の値には適用できない（-Inf や NaN を返す）。

第4章 データフレーム操作（読み込み・加工・保存）

📌 要点

  • tidyverse（readrなど）を使う現代的な書き方が授業推奨
  • パイプ演算子 |> でコードを上から下に読める
  • filter() で行の選択、select() で列の選択
  • mutate() で新しい列の追加・変換
  • pivot_longer() で横→縦変換、pivot_wider() で縦→横変換
  • 整然データ（tidy data）: 各変数＝列、各観測＝行、各値＝セル

📖 一般形

パッケージの導入
install.packages("パッケージ名")    # 初回のみ
library(パッケージ名)              # 毎回読み込み
データの読み込み
データ名 <- read_csv("ファイルパス.csv")
パイプ演算子
x |> f()    # f(x) と同じ意味
x |> f() |> g()    # g(f(x)) と同じ意味

%>% と |> は基本的に同じ。授業では |> を使用。

行の選択（filter）
データ |> filter(条件式)
データ |> filter(条件1 & 条件2)    # AND
データ |> filter(条件1 | 条件2)    # OR
データ |> filter(列名 %in% c("値1", "値2"))  # 複数値のいずれか
データ |> filter(!is.na(列名))    # NAを除外
列の選択（select）
データ |> select(列名1, 列名2)          # 指定列を残す
データ |> select(-c(列名1, 列名2))     # 指定列を除外
列の追加・変換（mutate）
データ |> mutate(
  新列名 = 計算式,
  新列名2 = as.factor(既存列),
  新列名3 = ifelse(条件, TRUEの値, FALSEの値)
)
ifelse関数
ifelse(条件, TRUEのときの値, FALSEのときの値)
列名の確認・変更
colnames(データ)              # 列名を確認
colnames(データ) <- 新しい名前のベクトル  # 列名を変更
データの形状変換
# 横→縦（ロング形式に変換）
データ |> pivot_longer(
  cols = 変換する列の指定,    # everything() で全列
  names_to = "新しい列名（元の列名が入る）",
  values_to = "新しい列名（値が入る）"
)

# 縦→横（ワイド形式に変換）
データ |> pivot_wider(
  names_from = 列名にする列,
  values_from = 値にする列
)
並び替え
データ |> arrange(列名)     # 昇順
データ |> arrange(desc(列名))  # 降順
重複の除去
データ |> distinct(列名)    # 指定列の一意な値を返す
値の取り出し
データ |> pull(列名)       # 列をベクトルとして取り出す
データ$列名                  # 同じ効果（Base R記法）
データ$列名[行番号]          # 特定の行の値
データフレームの結合
一般形説明
inner_join(x, y, by = "列名")共通の値が一致する行のみ結合
left_join(x, y, by = "列名")左側を全て保持し、右側をマッチ
right_join(x, y, by = "列名")右側を全て保持し、左側をマッチ
full_join(x, y, by = "列名")両方の全行を保持
データの保存
write_csv(データ, "パス/ファイル名.csv")    # CSVとして保存（型情報なし）
saveRDS(データ, "パス/ファイル名.rds")      # RDS形式で保存（型情報あり）
作業ディレクトリ・ファイル操作
getwd()                      # 現在の作業ディレクトリを確認
list.files("フォルダパス")     # フォルダ内のファイル一覧

⚠️ つまずきポイント

注意 列名にスペースが含まれる場合はバッククォートで囲む。
filter(`Country Name` == "Japan")
注意 CSVは型情報を保持しない。RDSは型情報を保持する。大きなデータではparquet形式が推奨。

第5章 データ集計（group_by・summarize・table）

📌 要点

  • group_by() |> summarize() はtidyverseでのグループ別集計の基本パターン
  • ダミー変数（0/1）の平均値 ＝ 条件を満たす割合
  • 論理条件の mean() も割合を返す（TRUE=1, FALSE=0）
  • table() はクロス集計表を作成
  • sapply() で複数列に関数を一括適用

📖 一般形

グループ別集計（最重要パターン）
データ |>
  group_by(グループ列) |>
  summarize(
    新列名 = mean(列名),    # 平均
    新列名 = sum(列名),     # 合計
    新列名 = median(列名)   # 中央値
  )
複数変数でグループ化
データ |>
  group_by(列名1, 列名2) |>
  summarize(新列名 = 集計関数(列名))
ダミー変数の平均 ＝ 割合（数式）

ダミー変数 $x_i \in \{0, 1\}$ について:

$$\bar{x} = \frac{1}{n}\sum_{i=1}^{n} x_i = \frac{k}{n}$$

$k$ は条件を満たす($x_i = 1$)データの数。つまり mean(call) = 書類通過率。

論理条件の割合を計算
mean(列名 == "値")        # "値"と一致する割合
mean(列名 < 数値)         # 数値未満の割合
mean(is.na(列名))        # 欠損の割合
sum(is.na(列名))         # 欠損の個数
クロス集計表
データ |>
  select(列名1, 列名2) |>
  table()                   # クロス集計

addmargins(集計表)          # 行・列の合計を追加
prop.table(集計表)          # 比率に変換

table(exclude = NULL)      # NAも集計に含める
sapply（複数列に関数を適用）
データ |>
  sapply(
    関数名,
    追加引数...
  )
十分位数の計算
データ |>
  select(列名1, 列名2) |>
  sapply(quantile, probs = seq(0, 1, by = 0.1), na.rm = TRUE)

🧮 具体例

例：人種別書類通過率 
resume |>
  group_by(race) |>
  summarize(
    call_rate = mean(call),   # 通過率
    call_count = sum(call)    # 通過数
  )
例：差の差分法（DID） 
did <- minwage |>
  group_by(state) |>
  summarize(
    fullBefore_prop = mean(fullBefore_prop),
    fullAfter_prop = mean(fullAfter_prop)
  ) |>
  mutate(diff_before_after = fullAfter_prop - fullBefore_prop)
# DID推定値:
did$diff_before_after[1] - did$diff_before_after[2]

⚠️ つまずきポイント

注意 group_by() は見た目を変えないが、以降の summarize() がグループごとに計算される。# Groups: の表示で確認可能。
注意 table() はデフォルトでNAを無視する。NAも集計に含めたい場合は table(exclude = NULL) を使う。

第6章 ggplotによるグラフ作成

📌 要点

  • ggplotの基本構造: ggplot(aes()) + geom_xxx() + labs() + theme()
  • グラフの種類: 棒グラフ(geom_bar)、ヒストグラム(geom_histogram)、箱ひげ図(geom_boxplot)、散布図(geom_point)、線グラフ(geom_line)
  • aes() でデータと見た目（x軸・y軸・色・形）の対応を指定
  • ggsave() でグラフを画像として保存

📖 一般形

ggplotの基本構造
データ |>
  ggplot(aes(x = x軸の列, y = y軸の列)) +
  geom_xxx() +
  labs(x = "x軸ラベル", y = "y軸ラベル", title = "タイトル") +
  theme_minimal(base_family = "フォント名")
棒グラフ
geom_bar(stat = "identity")    # y軸の値をそのまま使う

stat = "identity" を指定すると、データの値をそのまま棒の高さに使う。

ヒストグラム
ggplot(aes(x = 列名, y = after_stat(density))) +
geom_histogram(
  binwidth = ビンの幅,
  boundary = ビンの境界値,
  color = "枠線色",
  fill = "塗り色"
)

after_stat(density): y軸を密度にする。binwidth: 各ビンの幅。boundary: ビンの開始位置。

箱ひげ図
ggplot(aes(x = グループ列, y = 数値列)) +
geom_boxplot(
  fill = "塗り色",
  color = "枠線色",
  outlier.shape = 数値,    # 外れ値の形状
  outlier.size = 数値,     # 外れ値のサイズ
  outlier.alpha = 数値     # 外れ値の透明度
)

箱ひげ図の要素: 最小値、第1四分位数(Q1)、中央値、第3四分位数(Q3)、最大値。

散布図
ggplot(aes(x = 列名1, y = 列名2, color = グループ列, shape = グループ列)) +
geom_point(
  size = 点のサイズ,
  alpha = 透明度(0〜1)
)
線グラフ
ggplot(aes(x = 列名1, y = 列名2, color = グループ列, linetype = グループ列)) +
geom_line(linewidth = 線の太さ)
色・形状の手動設定
scale_color_manual(
  name = "凡例タイトル",
  labels = c("元の値" = "表示名", ...),
  values = c("元の値" = "色コード", ...)
)

scale_shape_manual(
  name = "凡例タイトル",
  values = c("元の値" = 形状番号, ...)
)

scale_color_brewer(palette = "パレット名")  # ColorBrewerのパレット
軸のカスタマイズ
scale_x_continuous(limits = c(最小, 最大), breaks = 目盛りベクトル)
scale_x_discrete(labels = c("ラベル1", "ラベル2", ...))  # 離散軸のラベル変更
グラフの保存
ggsave("フォルダ/ファイル名.png")

⚠️ つまずきポイント

注意 ggplotの各要素は + でつなぐ（パイプ |> ではない）。
注意 日本語フォントの指定: Macでは "Hiragino Sans"、Windowsでは "Yu Gothic" を使う。
注意 ggsave() はフォルダが存在しないとエラー。事前に dir.create("figure") が必要。

第7章 統計分析（相関・k平均法・Q-Qプロット）

📌 要点

  • 相関係数 cor(x, y) は2変数の線形関係の強さ（-1〜1）
  • Q-Qプロットは2つの分布の同一性を視覚的に確認する方法
  • $k$平均法は、データを$k$個のクラスターに分けるアルゴリズム
  • 対数変換 log() は分布の偏りを修正し、差を変化率に変換する

📖 一般形

相関係数
データ |>
  summarise(相関名 = cor(列名1, 列名2))
Q-Qプロット
qqplot(
  ベクトル1,
  ベクトル2,
  xlab = "x軸ラベル",
  ylab = "y軸ラベル",
  main = "タイトル",
  xlim = c(最小, 最大),
  ylim = c(最小, 最大)
)
abline(切片, 傾き, lwd = 線の太さ)  # 45度線: abline(0, 1)
k平均法
結果 <- データ |>
  select(変数1, 変数2) |>
  kmeans(centers = k, nstart = 試行回数)

結果$centers     # 各クラスターの中心点
結果$cluster     # 各データポイントのクラスター番号
クラスターと政党のクロス集計
table(
  party = データ$列名,
  cluster = 結果$cluster
)
自然対数変換
log(x)    # 底はe（ネイピア数）

対数変換の性質: $\log(x_2) - \log(x_1) = \log\left(\frac{x_2}{x_1}\right) \approx \frac{x_2 - x_1}{x_1}$（変化率への変換）

🧮 具体例

例：k平均法の実行と結果確認 
result <- congress |>
  filter(congress == 112) |>
  select(dwnom1, dwnom2) |>
  kmeans(centers = 2, nstart = 5)

result$centers    # 中心点の座標
table(
  party = congress$party[congress$congress == 112],
  cluster = result$cluster
)  # クラスターと政党の対応

⚠️ つまずきポイント

注意 $k$平均法のクラスター番号（1, 2, 3...）に意味はない。実行するたびに番号と中心点の対応が変わりうる。nstartで複数回試行し最良の結果を選ぶ。
注意 log(0) は -Inf、log(-1) は NaN を返す。対数変換する前にデータに0や負の値がないか確認すること。
注意 Q-Qプロットで点が45度線上にあれば2つの分布は同じ。線より上＝y側の方が大きい、線より下＝x側の方が大きい。

第8章 ループと条件文（for・if・else）

📌 要点

  • for (i in ベクトル) {} で同じ処理を繰り返す。i にベクトルの値が1つずつ順番に入る
  • 繰り返しの結果を保存するには、先に rep(NA, length(x)) で入れ物を作る
  • if (条件) {} else {} で条件によって処理を分岐する
  • 条件が3つ以上あるときは else if を使う。上から順に調べられ、最初に当てはまった処理だけ実行
  • %% は割り算の余りを返す演算子（x %% 2 == 0 なら偶数）
  • forとifは組み合わせて使える

📖 一般形

for構文（繰り返し）
for (i in ベクトル) {
  繰り返したい処理
}
結果を保存する繰り返し（入れ物パターン）
results <- rep(NA, length(values))   # 先に入れ物を作る

for (i in 1:length(values)) {
  results[i] <- 計算式              # 各回の結果を保存
}
if / else（条件分岐）
if (条件) {
  条件がTRUEのときの処理
} else {
  条件がFALSEのときの処理
}
else if（3つ以上の条件）
if (条件1) {
  処理1
} else if (条件2) {
  処理2
} else {
  処理3
}
剰余演算子
x %% 数値    # 割り算の余り。x %% 2 == 0 なら偶数

🧮 具体例

例：各要素を2倍して保存 
values <- c(2, 4, 6)
results <- rep(NA, length(values))

for (i in 1:length(values)) {
  results[i] <- values[i] * 2
}
results    # → 4 8 12
例：forとifの組み合わせ（偶数なら2倍、奇数なら2乗） 
values <- 1:5
results <- rep(NA, length(values))

for (i in 1:length(values)) {
  x <- values[i]
  if (x %% 2 == 0) {
    results[i] <- x * 2
  } else {
    results[i] <- x^2
  }
}
results    # → 1 4 9 8 25

⚠️ つまずきポイント

注意 if の中の条件は TRUEまたはFALSEのどちらか1つになる必要がある。
注意 else if は上から順に調べられ、最初に当てはまった処理だけが実行される。
注意 単純な計算なら values * 2 のようにベクトル演算で書く方が簡単なことも多い。for構文が役立つのは処理が複雑なときや各回の計算を順番に確認したいとき。

第9章 予測と予測誤差（選挙予測・バイアス・RMSE）

📌 要点

  • 予測誤差 ＝ 実際の結果 − 予測値
  • 予測誤差の平均値をバイアスと呼ぶ
  • 予測誤差を2乗して平均し平方根を取ったものが RMSE（2乗平均平方根誤差）。予測誤差の平均的な大きさを表す
  • 勝敗のようにカテゴリを予測する問題は分類問題。誤りは偽陽性・偽陰性に分けられる
  • as.Date() で文字列を日付型に変換すると、日付どうしの引き算で日数が計算できる
  • 予測と実際の符号（勝敗）の一致は sign() で確認できる

📖 一般形

予測誤差・バイアス・RMSE（数式）

$$\text{予測誤差} = \text{実際の結果} - \text{予測値}$$

$$\text{バイアス} = \text{予測誤差の平均値}, \qquad \text{RMSE} = \sqrt{\frac{1}{n}\sum_{i=1}^{n} \text{誤差}_i^2}$$

バイアスとRMSEの計算
データ |>
  summarise(
    bias = mean(誤差列),         # 平均予測誤差（バイアス）
    rmse = sqrt(mean(誤差列^2))   # 2乗平均平方根誤差（RMSE）
  )
日付の扱い
as.Date("YYYY-MM-DD")          # 文字列を日付型に変換
as.Date("日付") - 日付列       # 日付どうしの引き算で日数を計算
各州の直近調査から予測を作るパターン
データ |>
  mutate(margin = 列1 - 列2,
         days_to_election = as.Date("選挙日") - 日付列) |>
  group_by(グループ列) |>
  filter(days_to_election == min(days_to_election)) |>  # 直近の調査だけ残す
  summarise(pred = mean(margin), .groups = "drop")
符号（勝敗）の比較
sign(x)    # 正なら1、負なら-1を返す
データ |> filter(sign(予測) != sign(実際))    # 勝敗予測が外れた行を抽出
条件付き合計（選挙人票の集計）
sum(列名[条件])    # 条件を満たす行だけの合計。例: sum(EV[margin > 0])

🧮 具体例

例：2008年大統領選の予測誤差（結果） 
errors_df |>
  summarise(
    bias = mean(error),        # → 1.06
    rmse = sqrt(mean(error^2)) # → 5.91
  )
# バイアスはほぼ0 → 世論調査からの予測に大きな偏りはない
例：勝敗予測が外れた州 
errors_df |>
  filter(sign(pred_margin) != sign(actual_margin))
# → IN, MO, NC の3州のみ。いずれも接戦州だった

⚠️ つまずきポイント

注意 予測誤差は「実際 − 予測」の順。逆にすると符号（バイアスの向き）が反転する。
注意 偽陽性＝実際は陰性なのに陽性と予測、偽陰性＝実際は陽性なのに陰性と予測。向きを混同しやすい。
注意 散布図で予測（横軸）と実際（縦軸）を比べるとき、45度線上なら予測と実際が一致。右上・左下の領域は勝敗予測が的中、右下・左上は外れ。

第10章 線形回帰（feols・係数・予測値・残差・決定係数）

📌 要点

  • 線形回帰モデル: $Y = \alpha + \beta X + \epsilon$（$\alpha$＝切片、$\beta$＝傾き、$\epsilon$＝誤差項）
  • $\beta$ は「$X$ が1単位増加したとき $Y$ が平均してどれだけ増加するか」を表す
  • 授業では lm() ではなく fixestパッケージの feols() を使う（高速・高機能）
  • 係数は coef()、予測値は fitted()、残差は resid() で取り出す
  • 残差 ＝ 実際の値 − 予測値。残差のRMSEで予測精度を確認する
  • 決定係数 $R^2$ はモデルが説明した変動の割合（0〜1）。r2() で取得できる
  • 外れ値（例: パームビーチ郡）は回帰の当てはまりを大きく悪化させる

📖 一般形

線形回帰モデル（数式）

$$Y = \alpha + \beta X + \epsilon$$

推定値が得られれば、$\hat{Y} = \hat{\alpha} + \hat{\beta}X$ として予測値を計算できる。

feols()による推定
library(fixest)                # fixestパッケージを読み込む

fit <- feols(結果変数 ~ 予測変数, data = データ)
fit    # 推定結果を表示。切片は自動的にモデルに加えられる
推定結果の取り出し
一般形説明
coef(fit)切片と傾きの推定値
fitted(fit)予測値（当てはめ値）
resid(fit)残差（実際の値 − 予測値）
r2(fit, type = "r2")決定係数（fixestのr2関数）
predict(fit, newdata = データ)任意の予測変数の値に対する予測値
残差のRMSE
データ |>
  mutate(resid = resid(fit)) |>
  summarise(rmse = sqrt(mean(resid^2)))
決定係数（数式と手計算）

$$R^2 = \frac{\text{TSS} - \text{RSS}}{\text{TSS}} = 1 - \frac{\text{RSS}}{\text{TSS}}$$

TSS＝全変動平方和、RSS＝残差平方和。1に近いほどモデルが結果変数の変動をよく説明。

データ |>
  mutate(resid = resid(fit)) |>
  summarise(
    TSS = sum((結果変数 - mean(結果変数))^2),  # 全変動平方和
    RSS = sum(resid^2),                        # 残差平方和
    R2  = (TSS - RSS) / TSS                    # 決定係数
  )
回帰直線の描画
# 方法1: geom_smooth で回帰直線を描く
geom_smooth(method = "lm", se = FALSE)   # se = FALSE で信頼区間の帯を消す

# 方法2: 推定済みの係数から直線を描く
geom_abline(intercept = coef(fit)[1], slope = coef(fit)[2])
残差プロット
データ |>
  mutate(fitted = fitted(fit), resid = resid(fit)) |>
  ggplot(aes(x = fitted, y = resid)) +
  geom_point() +
  geom_hline(yintercept = 0)
結合時の共通列名の区別（suffix）
データ1 |>
  inner_join(データ2, by = "キー列", suffix = c("_2008", "_2012"))
# 両方にある列名（Obama, EVなど）に接尾辞を付けて区別する

🧮 具体例

例：顔の見た目と選挙結果の回帰 
fit <- feols(diff.share ~ d.comp, data = face)
coef(fit)    # → 切片 -0.312, 傾き 0.660
# 傾き0.66 → 能力スコアが0.1増えるとマージンが約6.6%ポイント増加
例：外れ値の影響（フロリダ州の回帰） 
fit2 <- feols(Buchanan00 ~ Perot96, data = florida)
r2(fit2, type = "r2")    # → 0.513

florida_nopb <- florida |> filter(county != "PalmBeach")
fit3 <- feols(Buchanan00 ~ Perot96, data = florida_nopb)
r2(fit3, type = "r2")    # → 0.851（外れ値を除くと大きく改善）

⚠️ つまずきポイント

注意 フォーミュラは「結果変数 ~ 予測変数」の順。逆にすると意味が変わる。
注意 相関係数（cor()）と回帰の傾き（$\hat\beta$）は別物。相関は関係の強さ（-1〜1）、傾きは$X$1単位あたりの$Y$の変化量。
注意 $R^2$ が高くても予測が正確とは限らない。見た目スコアの回帰では残差のRMSEが約0.26（26%ポイント）もあり、予測はそれほど正確ではない。

第11章 回帰分析と因果推論（実験・重回帰・交互作用）

📌 要点

  • 2値トリートメントの実験データでは、回帰の傾き係数 ＝ 平均トリートメント効果の推定値 ＝ 2グループの平均の差
  • この因果解釈にはトリートメントの割り当てがランダムであるなどの仮定が重要
  • 重回帰モデル: $Y = \alpha + \beta_1 X_1 + \beta_2 X_2 + \cdots + \beta_p X_p + \epsilon$。各係数は「他の予測変数を一定に保ったとき」の関係
  • 因子変数（文字列の変数）は自動的にダミー変数に変換されて回帰に入る
  • X1 * X2 と書くと主効果と交互作用項がまとめてモデルに入る
  • 交互作用項の係数は、グループによってトリートメント効果がどれだけ異なるか（不均一トリートメント効果）を表す

📖 一般形

トリートメント効果の回帰（2値トリートメント）
feols(結果変数 ~ トリートメント変数, data = データ)
# トリートメント変数の係数 = 平均トリートメント効果の推定値
# = トリートメントグループとコントロールグループの平均の差
重回帰モデル（数式）

$$Y = \alpha + \beta_1 X_1 + \beta_2 X_2 + \cdots + \beta_p X_p + \epsilon$$

各係数 $\beta_j$ は、他の予測変数を一定に保ったときに $X_j$ が1単位増加することと $Y$ の平均的な変化の関係を表す。

因子変数の回帰（自動ダミー変換）
feols(結果変数 ~ 因子変数, data = データ)
# 因子変数は自動的に指標変数（ダミー変数）に変換される
# 基準カテゴリ以外の各カテゴリに係数が付く
predict()による予測値の計算
newdata <- データ |> distinct(列名)   # ユニークな値のデータフレーム
predict(fit, newdata = newdata)       # 各値に対する予測値
# この予測値は各グループの実際の平均と一致する
交互作用項を含む回帰
feols(結果変数 ~ 変数1 * 変数2, data = データ)
# 変数1、変数2の主効果と交互作用項（変数1:変数2）がまとめて入る

$$Y = \alpha + \beta_1 X_1 + \beta_2 X_2 + \beta_3 (X_1 \times X_2) + \epsilon$$

$\beta_3$（交互作用項の係数）は、$X_2$ のグループによって $X_1$ の効果がどれだけ異なるかを表す。

🧮 具体例

例：インドの女性割当のランダム化実験 
feols(water ~ reserved, data = women)
# reservedの係数 9.25 = 女性割当の平均因果効果（飲用水設備が増加）
feols(irrigation ~ reserved, data = women)
# 係数 -0.37 で有意でない（灌漑設備には効果なし）
# → 女性政治家は女性有権者の求める政策を支持しやすいという仮説と整合的
例：社会的プレッシャー実験の交互作用 
fit_int <- feols(primary2006 ~ primary2004 * messages,
                 data = social_neighbor)
# 交互作用項 primary2004:messagesNeighbors の係数 0.027
# → 2004年に投票した人ほどNeighborsメッセージの効果が大きい
#   （不均一トリートメント効果）

⚠️ つまずきポイント

注意 回帰係数を因果効果と解釈できるのは、トリートメントの割り当てがランダムなどの仮定が満たされる場合だけ。観察データの回帰は基本的に予測の道具。
注意 X1 * X2（主効果＋交互作用）と X1 : X2（交互作用項のみ）は別物。通常は * を使う。
注意 因子変数の回帰では基準カテゴリが切片に吸収される。各係数は基準カテゴリとの差を表す。
